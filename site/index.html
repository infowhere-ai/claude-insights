What it does

Everything Claude Code doesn't show you — shown.

All data stays on your machine. No accounts, no telemetry, no cloud.

🟢

Live Status

Current state in real time — working, waiting for input, compacting, idle. Shows the exact tool name as it executes.

🧠

Reasoning Visibility

Claude's internal thinking streamed live as it arrives. Full reasoning history with word count and block numbers, browsable across the entire session.

📊

Context Window

Full breakdown of what's consuming your context — fixed rules, conversation, and accumulated tool results, with token costs per file.

🪙

Token Usage

Input, output, and cache reads per session. Tracks the 5-hour renewal window so you always know how much time is left.

⚡

Command History

Every tool call logged with path, duration, and success or failure. See exactly what Claude touched and how long each step took.

🤖

Agent Tracking

Sub-agents and parallel tasks tracked separately. Duration, status, and which parent session launched them — all in one panel.

📂

Git Diff Viewer

Uncommitted changes at a glance. Click any file for an IntelliJ-style side-by-side diff with the reasoning that caused it.

🗂️

Multi-Project

Auto-discovers every directory with a .claude/ folder under your root. Switch projects with one click — no config needed.

🕐

Session History

Browse and replay past sessions — tool calls, reasoning blocks, and context breakdowns — while a live session runs in parallel.

📱

Mobile Responsive

Works on any screen size. Open the dashboard on your phone or tablet while Claude works on your desktop — panels reflow cleanly.

Live Status

Every keystroke.
Every tool. Live.

The status bar tells you exactly what Claude is doing at this instant — the tool name, the file, the state. Not "working". Working — reading static/insights.html. The difference matters when you're trying to understand why something is taking long.

The LIVE badge in the header confirms real-time monitoring is active. When Claude Code is idle, the badge switches to HISTORY mode — the dashboard keeps working, showing data from past sessions.

  • ✓ Real-time status: WORKING · IDLE · COMPACTING · WAITING
  • ✓ Exact tool name shown as it executes
  • ✓ LIVE / HISTORY badge with automatic switching
  • ✓ Green pulsing indicator — no refresh needed
Git diff viewer

Your IDE's diff view.
In the browser.

Claude Code changes dozens of files per session. Claude Insights lets you review every uncommitted change side-by-side, with syntax highlighting — without leaving the browser.

But that's only half of it. Below every diff, a Change Summary tells you exactly what caused it: how many lines changed, which tool did it, how many times — and the reasoning Claude was following when it happened.

  • ✓ Side-by-side diff with syntax highlighting
  • ✓ Change Summary: +N added · -N removed · function name
  • ✓ Which tool triggered the change and how many times
  • ✓ Claude's reasoning that led to the change — below the diff
Reasoning

Claude thinks out loud.
You watch live.

Claude's internal reasoning is normally invisible. Every thought block streams to you in real time as Claude works — you see the plan before the tool call happens. You'll catch wrong assumptions before they cost you ten minutes.

Every reasoning block across the entire session is saved and browsable. 123 blocks? All there, collapsed by default, expandable on click. Word count, block number, full text — nothing gets lost.

  • ✓ Live reasoning streamed word by word
  • ✓ Block number, word count, timestamp per block
  • ✓ Full history modal — all blocks from the session
  • ✓ Works across sessions — history browsable even for idle projects
Context window

Know exactly what's
eating your context.

Every token counts. Claude Insights breaks your context window into three segments: fixed rules loaded at session start, the growing conversation, and accumulated tool results. Token costs shown per file — so you know what to trim.

The model panel tracks input, output, and cache reads in real time. An 80% bar means context pressure is building — you'll see it before Claude hits the wall and starts compacting.

  • ✓ Three-segment breakdown: FIXED · CHAT · VARIABLE
  • ✓ Per-file token cost in the FIXED segment
  • ✓ Cache hit tracking — see how much you're saving
  • ✓ Context pressure bar with colour-coded urgency
Multi-project

All your projects.
One dashboard.

Claude Insights auto-discovers every directory that has a .claude/ folder under your configured root. No config files, no manual registration — if Claude Code knows about it, the dashboard knows about it.

Switch between projects with a single click. The dashboard reloads instantly with the new project's live session, history, and stats. Nine projects in the dropdown? Still one click.

  • ✓ Auto-discovery — zero configuration required
  • ✓ Instant switch — no page reload
  • ✓ Live and History modes per project independently
  • ✓ Works with any number of projects simultaneously
Git integration

Every file Claude touched.
Before you commit.

The To Commit panel lists every uncommitted change in your project, updated live as Claude works. Modified files, new files, staged changes — all at a glance, without switching to a terminal.

Click any file to open the diff viewer. Review what actually changed, see the reasoning that caused it, and decide what goes into the commit. No git status, no git diff, no context switch.

  • ✓ Live uncommitted file list — updates as Claude writes
  • ✓ M / ? / A status indicators
  • ✓ One click to open full diff
  • ✓ Reasoning context preserved next to each change
Commands & Agents

Every tool call.
Every agent. Tracked.

The Commands panel logs every tool call in chronological order — tool name, file path, duration, success or failure. A typical session has hundreds of calls. You'll know which ones took 3 seconds and which ones failed silently.

Sub-agents get their own panel. Duration, status, which session launched them — tracked separately so parallel work doesn't get lost in the main timeline.

  • ✓ Full command history: name · path · duration · result
  • ✓ Tool icons for instant visual scanning
  • ✓ Agent panel: sub-agents tracked independently
  • ✓ Duration per agent — spot bottlenecks in parallel work
Session history

Yesterday's session.
Still there.

When Claude Code isn't running in a project, the dashboard switches to HISTORY mode automatically. Everything from past sessions — tool calls, reasoning blocks, context breakdowns — is still browsable.

The Activity panel shows aggregate stats: total sessions, tokens over 7 days, top tools. Use it to understand how you work, not just what Claude is doing right now.

  • ✓ Automatic LIVE / HISTORY mode switching
  • ✓ Navigate past sessions while live session runs
  • ✓ Activity stats: sessions, tokens, top tools
  • ✓ 7-day token trend at a glance
Usage & Activity

The 5-hour clock.
Your usage trends.

Claude Code resets token limits every 5 hours. Most people discover this mid-session when things stop working. Claude Insights shows you exactly how many tokens you've used, how many sessions are active, and how much time is left in the current window.

The Activity panel aggregates across all sessions: total token spend over 7 days, top tools, session count. Build intuition for how your work patterns translate to usage.

  • ✓ 5-hour window progress bar with time remaining
  • ✓ Tokens consumed + sessions active in current window
  • ✓ 7-day aggregate: tokens, sessions, top tools
  • ✓ Plan your work around the reset, not the surprise

How it works

Hooks + SSE. That's it.

No cloud services, no background daemons, no accounts. A shell script and a local server.

Claude Code → hook fires → .claude/status.json → claude-insights (SSE) → browser
1

Claude Code fires hooks

Before and after every tool call, on notifications, on stop — Claude Code calls monitor-hook.sh automatically.

2

Hook writes a JSON file

The script writes .claude/status.json inside the current project with the current state, tool, and context.

3

Server streams updates

Claude Insights polls the project directories and pushes changes to the browser via Server-Sent Events — no polling from the client.

Quick start

1
Activate hooks

Sets up ~/.claude/hooks/monitor-hook.sh and registers it for 5 Claude Code events. Existing hooks are never overwritten.

claude-insights install
2
Restart Claude Code

Required for the hooks to activate in any open sessions.

3
Start the dashboard

Opens at localhost:4000

claude-insights start
